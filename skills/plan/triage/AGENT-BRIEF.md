# Agent Brief

A comment posted on the issue that tells an AFK agent exactly what to do. The agent has no conversation context — this brief is everything they'll see.

## Structure

```markdown
> *This was generated by AI during triage.*

## What to Build

[Concise description of the vertical slice. Describe end-to-end behavior, not layer-by-layer implementation. Use domain glossary vocabulary.]

## Acceptance Criteria

- [ ] Criterion 1
- [ ] Criterion 2
- [ ] Criterion 3

## Relevant Context

- Key files / modules to touch (if known)
- ADRs that constrain the implementation
- Domain terms to use (from CONTEXT.md)
- Test patterns to follow (reference existing tests)

## Blocked By

- Reference to blocking ticket, or "None — can start immediately"
```

## Rules

- **Use domain language.** Every term comes from CONTEXT.md. No generic CS jargon.
- **Avoid file paths when possible.** They go stale. Reference modules and concepts instead.
- **Be specific about acceptance.** "Works correctly" is not a criterion.
- **Include test guidance.** Reference prior art tests in the codebase.
- **Keep it concise.** The agent has a fresh session — don't repeat obvious conventions.
